# 东软颐养中心管理系统 — 功能规格说明书

---

## 1. 系统概述

本系统是面向养老机构的综合管理平台，支持客户入住、床位调度、膳食安排、护理管理、服务跟踪等核心业务的数字化运营。系统采用前后端分离架构，支持多角色权限控制。

---

## 2. 用户角色

| 角色 | 代号 | 描述 | 典型用户 |
|------|------|------|----------|
| 系统管理员 | ROLE_ADMIN | 拥有全部功能权限，可管理系统用户 | 机构管理层 |
| 健康管家 | ROLE_MANAGER | 管理客户、床位、入住、膳食、护理等业务 | 护理部门主管 |
| 护士 | ROLE_NURSE | 执行护理操作，记录护理结果 | 一线护理人员 |
| 入住老人 | ROLE_USER | 查看个人信息、膳食安排、外出登记 | 入住客户 |

### 3.4 入住登记

**功能描述**：办理客户入住，自动关联床位。

| 功能 | 说明 |
|------|------|
| 入住列表 | 分页展示，显示登记编号、客户姓名、床位号、入住日期 |
| 新增入住 | 选择客户 + 床位，填写入住信息 |
| 入住详情 | 抽屉展示详细信息 |
| 删除入住 | 逻辑删除 |

**业务规则**：
1. 选择客户后，系统自动筛选空闲床位
2. 提交后自动生成登记编号：`RZ{yyyyMMdd}-{序号}`
3. 自动更新床位状态为「已入住」，关联客户 ID
4. 自动更新客户 `checked_in = 1`
5. 三表联动，事务保证一致性

**表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 客户姓名 | 下拉 | ✓ | 从客户列表选择 |
| 床位 | 下拉 | ✓ | 从空闲床位选择 |
| 入住日期 | 日期 | ✓ | |
| 合同期限 | 数字 | | 单位：月 |
| 押金 | 数字 | | 单位：元 |
| 经办人 | 下拉 | | 从用户列表选择 |

### 3.5 退住登记

**功能描述**：办理客户退住，自动释放床位。

| 功能 | 说明 |
|------|------|
| 退住列表 | 分页展示，显示退住编号、客户姓名、退住日期、原因 |
| 新增退住 | 选择客户，填写退住信息 |
| 删除退住 | 逻辑删除 |

**业务规则**：
1. 只能选择已入住的客户
2. 提交后自动生成退住编号：`TZ{yyyyMMdd}-{序号}`
3. 自动查询入住记录获取床位 ID
4. 释放床位：状态改为空闲，清空客户关联
5. 更新客户 `checked_in = 0`

**表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 客户姓名 | 下拉 | ✓ | 从已入住客户选择 |
| 退住日期 | 日期 | ✓ | |
| 原因 | 多行文本 | | |
| 经办人 | 下拉 | | |

## 4. 非功能需求

| 需求 | 说明 |
|------|------|
| 响应时间 | 页面加载 < 2 秒，接口响应 < 1 秒 |
| 并发支持 | 支持 50+ 用户同时在线 |
| 数据安全 | BCrypt 密码加密，JWT 认证，SQL 注入防护 |
| 逻辑删除 | 所有删除操作为逻辑删除，数据可恢复 |
| 文件存储 | 图片上传到阿里云 OSS，CDN 加速 |
| 浏览器兼容 | Chrome 90+、Edge 90+、Firefox 90+ |
